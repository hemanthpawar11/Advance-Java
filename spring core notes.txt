Dependency--is nothing but datamembers.
Dependency injection--
-It is a design pattern to inject the dependency(initialize the datamembers).
-It is a technique in which an object receives other objects that it depends on.
-It is fundamental aspect of spring framework.

for example:-----Suppose class Car is dependent on class Engine,
Now this can creates many problems in real world applications including system failure.
Hence such dependencies needs to be avoided.
Spring resolves such dependencies using dependency injection.
Here it indicates that  class engine will be injected into class Car by spring.
Dependency injection makes the code easy to test and reuse.
While creating complex java application the classes should be independent of each other to improve possibility of reusing this classes and to test them independently.
Dependency injection allows these classes to be together and at the same time keeps them independently.

IOC
-IOC stands for Inversion of control--(control of object creation is given to spring itself rather than programmer creating the object hence called as Inversion of Control).
-It is a process where spring framework will inject one object into another object (and it is done automatically by spring container at runtime dynamically). 
-Thus ,IOC makes The code loosely coupled.In such case ,there is no need to modify the code if our logic is moved to new environment.
-In spring framework ,IOC container is responsible for injecting dependency.

--They are of different types, one among them is Spring IOC container.
--Key feature of IOC Container is Dependency Injection.
--The spring IOC container is the core of spring framework.
--IOC container will read the xml file/config class.
--It creates object ,configure and manage the entire life cycle of that object.
--The IOC container uses Dependency Injection to manage the components of the application.

Dependency injection using @Value annotation

Pros:
Integrated with Spring’s dependency injection system.
Automatically pulls config values from external sources (properties/env).
Good for centralized, static configuration (e.g., port, URLs, credentials).

Cons:
Not dynamic: Values are fixed when the application starts.
You can’t easily change the values at runtime.
Only one instance managed by Spring (singleton by default).

@Value
It is method/datamember level annotation.
It is used to inject primitive dependencies.

1.Directly on Datamember
ex:
@Value(value="10)
int age;

2.Setter DI
ex:
int age;
	Person(@Value(value = "10") int age){
		this.age=age;
	}

	public int getAge() {
		return age;
	}

3.Constructor DI
ex:
	@Value(value = "10")
	public void setAge(int age) {
		this.age = age;
	}

Scope of bean
It is single bean definition made to the spring bean.

singleton-If the scope is singleton then spring container will create only one instance of bean and same instance is used for all request of that bean.

prototype-If the scope is prototype then spring container will create 
new instance for every request made.

@Scope
It is class level annotation.It must be used along with @Component.
It specifies the scope of objects for the specified class.
By default, the spring framework creates exactly one object for component class,which means by default component class is a singleton class.
If we want to change the scope of the class to allow multiple objects to be created ,then we need to set the scope value as prototype.
@Scope(value="prototype")

Interface dependency Injection
It is a type of dependency injection where interface is used to inject the dependencies (implementation class of an interface is used to inject the dependency).

Steps:
Create an interface
Create implementation class
Create spring config class
Create Driver class

---------------------------------------------------

When we have reference dependency of interface and used @Autowired on it,Spring container will create object of its impementation class and do the dependency injection.

When we have 2 implementation class for same interface here it will lead to NoUniqueBeanDefinitionException.

To avoid this we use @Primary annotation on any one of the implementation class.

@Primary is class level annotation used to prioratize one implementation class to inform spring container to inject object to interface type dependency. 
------------------------------------------------------------
@Bean

it is method level annotation.

Hence in config class we create a method which returns me that predefined class
object.On that method we specify @Bean.
So the spring container will understand to take object from config class and do the dependency injection.

---------------------------------------------------------
Loose Coupling in Spring 
What is Coupling?
Definition
Coupling means the degree of dependency between two classes or modules.
It shows how strongly one class is connected to another class.

Types of Coupling
There are two types:
1.Tight Coupling
2.Loose Coupling

Tight Coupling
Definition
When one class directly creates and depends on another class, it is called Tight Coupling.

Example (Without Spring)
class Mobile {
    void call() {
        System.out.println("Calling...");
    }
}

class Customer {
    Mobile mob = new Mobile();   // tightly coupled

    void usePhone() {
        mob.call();
    }
}

Problems
Cannot change Mobile easily
Hard to test
Hard to maintain
Code is rigid
Reusability is low

Suppose you want:
Change Mobile → iPhone
You must modify:Customer class
Not good design

Loose Coupling
Definition
When classes depend on abstraction (interface) instead of concrete implementation, it is called Loose Coupling.
Objects are independent.

Example (Using Interface)
interface Mobile {
    void call();
}

class Samsung implements Mobile {
    public void call() {
        System.out.println("Samsung calling");
    }
}

class Nokia implements Mobile {
    public void call() {
        System.out.println("Nokia calling");
    }
}

Now Customer
class Customer {
    Mobile mob;

    Customer(Mobile mob) {
        this.mob = mob;
    }
}

No new keyword
Depends only on interface
With Spring (Best Example)
@Component
class Samsung implements Mobile { }

@Component
class Customer {

    @Autowired
    Mobile mob;   // Spring injects dependency
}

What happens internally?
Spring:
Creates Samsung object
Creates Customer object
Injects Samsung into Customer

Flow Diagram
Customer  →  Mobile (interface)
                 ↑
        Samsung / Nokia

Spring chooses and injects

Tight vs Loose Coupling Comparison
Feature		Tight Coupling	Loose Coupling
Dependency	Direct class	Interface
Flexibility	Low		High
Maintenance	Difficult	Easy
Testing		Hard		Easy
Reusability	Poor		Good
Spring DI	Not used	Used

Real-Time Example:
Tight Coupling
Car → PetrolEngine
If engine changes → Car must change 

Loose Coupling
Car → Engine (interface)
PetrolEngine / DieselEngine / ElectricEngine
Change engine easily 

How Spring Achieves Loose Coupling?
Spring uses:
1. IOC
Spring creates objects
2. Dependency Injection
Spring injects objects
3. Interfaces
Depend on abstraction
4. @Autowired
Automatic wiring

Key Interview Definition
Short Answer:Loose Coupling means reducing dependency between classes by using interfaces and dependency injection.
One-line Answer:Classes depend on interfaces, not implementations.

Advantages of Loose Coupling
Easy to change implementation
Easy testing (mock objects)
Easy maintenance
Reusable code
Flexible design
Better scalability

3. What is IOC (Inversion of Control)?
Definition
IOC means:Object creation and management is done by Spring, not by the programmer.

Without IOC (Traditional Java)
Mobile m = new Mobile();
Customer c = new Customer(m);

Programmer creates objects manually → Tight coupling
With IOC (Spring)
@Autowired
Mobile mob;

Spring creates and injects objects automatically → Loose coupling 

Benefits of IOC
Reduces dependency
Easy maintenance
Easy testing
Code becomes flexible

6. Ways to Create Beans
Method 1 → @Component
Meaning:Automatically creates bean using component scanning.

Example
@Component
public class Mobile { }

Used for
Our own classes

Method 2 → @Bean
Meaning:Manual bean creation inside configuration class.

Example
@Bean
public Scanner scanner() {
    return new Scanner(System.in);
}

Used for
Third-party classes
Inbuilt classes
Collections
When source code cannot be modified

Difference: @Component vs @Bean
Feature		@Component	@Bean
Type		Automatic	Manual
Location	Class		Method
Use		Our classes	External classes

10. @Autowired

13. @Qualifier
Definition:Specifies exact bean name.

Example:

-------------------------------------------
14. @Primary
Definition:Marks default bean among multiple beans.

Example:

@Primary
@Component
public class Samsung implements Mobile

Meaning:If multiple Mobile beans exist → Samsung is default

Difference: @Primary vs @Qualifier
Feature		@Primary	@Qualifier
Selection	Default		Specific
Usage		Class level	Field level
