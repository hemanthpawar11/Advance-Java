Spring MVC
A spring mvc is sub framework of of spring framework.
It is built on top of servlet api.
It follows Model-View-Controller design pattern.
It implements all the basic features of core spring framework like inversion of control and dependency injection.
A spring mvc provides an elegant solution to use MVC in spring framework by the help of DispatcherServlet.


MVC pattern stands for Model-View-Controller pattern.

--Model - A model contains the data of the application. A data can be a single object or a collection of objects.
--Controller - A controller contains the business logic of an application. Here, the @Controller annotation is used to mark the class as the controller.
--View - A view represents the provided information in a particular format. Generally, JSP is used to create a view page. 
--Front Controller - In Spring Web MVC, the DispatcherServlet class works as the front controller. It is responsible to manage the flow of the Spring MVC application.
 DispatcherServlet is a class that receives the incoming request and maps it to the right resource such as controller,models and views
DispatcherServlet that handles all the Http requests and responses.

Why spring mvc
-Separate roles - The Spring MVC separates each role, where the model object, controller, command object, view resolver, DispatcherServlet, validator, etc. can be fulfilled by a specialized object.
-Light-weight - It uses light-weight servlet container to develop and deploy your application.
-Powerful Configuration - It provides a robust configuration for both framework and application classes.
-Rapid development - The Spring MVC facilitates fast and parallel development.
-Reusable business code - Instead of creating new objects, it allows us to use the existing business objects.
-Easy to test - In Spring, generally we create JavaBeans classes that enable you to inject test data using the setter methods.
-Flexible Mapping - It provides the specific annotations that easily redirect the page.

How Spring web mvc works:

-Initially all the incoming request is intercepted by the DispatcherServlet that works as the front controller.
-The DispatcherServlet gets an entry of handler mapping from the XML file and forwards the request to the controller.
-The controller returns an object of ModelAndView.
-The DispatcherServlet checks the entry of view resolver in the XML file and invokes the specified view component.

(Annotation Configuration):
How Spring Web MVC works 

Initially, all incoming HTTP requests are intercepted by the DispatcherServlet, which acts as the Front Controller.
The DispatcherServlet uses @RequestMapping annotations to identify the appropriate controller method to handle the request.
The controller method processes the request and typically returns a ModelAndView object or a logical view name along with model data.
The DispatcherServlet consults a ViewResolver bean (configured via Java-based configuration using @Bean in a @Configuration class) to resolve the logical view name into an actual view (e.g., JSP).
The resolved view is rendered, and the final response is sent back to the client.

Spring MVC – Notes Q&A

1. Introduction to Spring MVC
Spring MVC is a Java-based web framework developed as a part of the Spring Framework to build robust, 
scalable, and loosely coupled web applications.
It follows the MVC (Model–View–Controller) design pattern, which separates the application into 
three independent components for better maintainability and scalability.

It separates:
Model → Data (Java objects)
View → JSP/HTML pages
Controller → Handles requests

MVC Architecture
Browser
   ↓
DispatcherServlet (Front Controller)
   ↓
Controller
   ↓
Model/Data
   ↓
View (JSP)
   ↓
Response to Browser

4. Reading Form Data Methods
Your project shows 3 ways to read form data.

Method 1 – HttpServletRequest (Old way)
String name = req.getParameter("username");

Problems:Manual conversion,Not recommended

Method 2 – @RequestParam
public String readData(
    @RequestParam("username") String name,
    @RequestParam("userage") int age)

Advantages:Cleaner,Type conversion automatic.

Method 3 – @ModelAttribute (BEST – used in your code)
@RequestMapping("/read")
public String readData(@ModelAttribute Person p)

What happens internally?
Spring:
Creates Person object
Reads form fields
Calls setters
Populates object automatically

Your Model
Person.java
class Person {
    String username;
    int userage;
    long userphono;
    char usergender;
}

Matching rule
Form input name must match variable name:
username → setUsername()
userage → setUserage()

Advantages:Clean code,Automatic binding,Object oriented,Recommended

10. Annotations in Spring MVC
Spring MVC uses annotation-based configuration to reduce XML.

@Controller:Marks class as controller
@RequestMapping:Maps URL to method
@RequestParam:Binds single request parameter to variable
@ModelAttribute:Binds form data to object automatically
@Configuration:Defines configuration class
@ComponentScan:Scans packages for beans
@Autowired:Performs dependency injection

11. Data Binding in Spring MVC
Definition
Data Binding is the process of converting HTTP request parameters into Java objects automatically.

Types
1. Manual Binding
Using HttpServletRequest
(Not recommended)
2. @RequestParam:For single value
3. @ModelAttribute:For object binding (Best approach)

Benefits
 Reduces boilerplate code
 Automatic conversion
 Clean and readable
